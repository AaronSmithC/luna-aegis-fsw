## 01 — CHANGE SUMMARY

This ECL extends the Short Hopper FSW with a NAV trajectory model, LUNET
refuel/recharge sequencing, sequence gate enforcement in MM, an orbital
propellant depot at Aegis Station, and a ground console overhaul for
mission planning.

**A. NAV Trajectory Model** — Simplified 1D range model. Distance-to-destination
initialized from the destination table at mission start and decremented by
integrated speed during powered/coast phases. NAV_StatePkt extended with
DistToDest_m and DestId fields. NAV SET_DEST command (FC=2) sets the target
destination. Origin tracking enables multi-hop distance computation (not
always from Shackleton).

**B. LUNET Refuel/Recharge Sequence** — 5-state FSM: IDLE → HANDSHAKE (3s) →
CARTRIDGE_MATE (5s) → PROP_TRANSFER (until tanks full) → RECHARGE (until
SOC ≥ 95%) → COMPLETE. Triggered on LANDED or DOCKED at a refuel-capable
destination. Auto-refuel toggle (LUNET_AUTO_REFUEL_CC, FC=3) allows manual
control. Manual trigger via LUNET_REFUEL_START_CC (FC=2). LUNET reads
DestId from NAV to simulate the correct LUNET beacon node.

**C. PROP Refuel Command** — PROP_REFUEL_CC (FC=6) fills LOX/LH2 at 50 kg/s
maintaining 6:1 O/F ratio. Caps at full capacity (2062.5 kg LOX + 687.5 kg
LH2). Safety interlock: refueling stops on engine start.

**D. Sequence Gate Enforcement** — SeqGate_Check() wired into MM_TransitionTo.
Every non-ABORT/SAFED transition runs through the gate table. Failed gates
emit EVS error with the blocking prereq name and reject the transition.
Gate table updated: prereqs only check pre-transition state (removed
ENGINE_RUNNING from PREFLIGHT→POWERED_ASC, etc.). Added orbital profile
gates (HOVER→RENDEZVOUS, RENDEZVOUS→DOCKING, UNDOCKING→POWERED_ASC).

**E. Aegis Station Orbital Depot** — Aegis Station (dest ID 1) upgraded to
refuel-capable with LUNET node ID 8. Refuel sequence triggers on DOCKED
phase. Enables round-trip orbital missions (1680 m/s one-way, refuel at
station, 1680 m/s return).

**F. Ground Console Overhaul** — Mission planning with current-location tracking,
hop-to-hop Haversine distances, Δv budget check (one-way for refuel sites,
round-trip otherwise). Destination dropdown matches la_destinations.h.
Live distance-to-destination in NAV panel. Refuel/recharge status indicators
in PROP and POWER panels. Auto-refuel ON/OFF toggle + REFUEL NOW button.

**G. Bridge Telemetry Fixes** — Replaced FIFO queue with latest-value table
(per-type dict) to eliminate priority inversion between high-rate and low-rate
MIDs. Broadcast loop sweeps all dirty entries at 10 Hz — every MID type gets
delivered regardless of traffic volume. Unparsed cFE core MIDs dropped at recv.
SO_RCVBUF increased to 1 MB. NAV decimated to 2 Hz, PROP decimated to 2 Hz.
Seven unparsed high-rate MIDs removed from TO_LAB subscription table
(IMU, ACS, ALT, GDN, TRN, TCS, LC). LUNET beacon parser added.

**H. Build System** — install.sh copies fsw/common/*.h to la_common include dir.
Include rewriting extended to .h files in app src dirs. TO_LAB high-rate
MID removal applied as sed patch in install.sh. TO_LAB MsgLim values increased
via sed patch in install.sh (10 Hz MIDs: 4→16, 1 Hz MIDs: 4→8, cFE core: 4→8).

**I. SB Pipe Overflow Root Cause** — Linux `fs.mqueue.msg_max` kernel default (10)
silently capped all POSIX message queues to 10 messages regardless of the
256 depth requested by cFS. With 16 subscribed MIDs on the TO_LAB TLM pipe,
the queue filled instantly at startup and remained saturated during operation.
High-rate NAV (10 Hz) monopolized the 10 available slots, starving 1 Hz MIDs
(LG, DOCK, HS). Fix: raise kernel limit to 256 (`sysctl -w fs.mqueue.msg_max=256`).
Console font loading changed from blocking @import to system fonts (no external
dependencies).
